## 部署步骤

### 1. 准备部署文件

**vercel.json**
```json
{
  "rewrites": [
    { "source": "/(.*)", "destination": "/src/index.html" }
  ],
  "headers": [
    {
      "source": "/(.*)",
      "headers": [
        { "key": "Cache-Control", "value": "public, max-age=3600" },
        { "key": "Service-Worker-Allowed", "value": "/" }
      ]
    }
  ]
}
```

**package.json**
```json
{
  "name": "workplace-power-personality-test",
  "version": "1.0.0",
  "description": "职场权力人格测评H5应用",
  "scripts": {
    "start": "serve src -p 3000",
    "build": "echo 'Static site, no build needed'"
  },
  "dependencies": {
    "serve": "^14.2.1"
  },
  "engines": {
    "node": ">=18"
  }
}
```

### 2. 部署到 Vercel

**方法一：通过 Vercel CLI（推荐）**
```bash
# 安装 Vercel CLI
npm i -g vercel

# 登录
vercel login

# 部署
vercel
```

**方法二：通过 GitHub 自动部署**
1. 将代码推送到 GitHub 仓库
2. 访问 vercel.com
3. 导入 GitHub 仓库
4. 自动部署

### 3. 部署到 Netlify（备选）

**netlify.toml**
```toml
[build]
  publish = "src"
  command = "echo 'No build needed'"

[[redirects]]
  from = "/*"
  to = "/index.html"
  status = 200

[[headers]]
  for = "/*"
  [headers.values]
    Cache-Control = "public, max-age=3600"
    Service-Worker-Allowed = "/"
```

### 4. GitHub Pages 部署

**方法一：通过 gh-pages**
```bash
# 安装 gh-pages
npm i -D gh-pages

# 部署
npm run deploy
```

**package.json 添加：**
```json
"scripts": {
  "predeploy": "npm run build",
  "deploy": "gh-pages -d src"
}
```

**方法二：手动部署**
1. 将 `src/` 目录内容推送到 `gh-pages` 分支
2. 在 GitHub 仓库设置中启用 GitHub Pages

## 故障排除

### 问题：Service Worker 不更新
**解决方案：**
1. 更新 service-worker.js 中的版本号
2. 用户关闭所有标签页重新打开

### 问题：图标不显示
**解决方案：**
1. 检查图标文件路径是否正确
2. 确保图标文件已上传
3. 清除浏览器缓存

### 问题：PWA 无法安装
**解决方案：**
1. 确保 HTTPS
2. 检查 manifest.json 配置
3. Service Worker 必须注册成功
